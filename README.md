## ðŸš€ Overview

The analysis script (analyze_genimage.py) automatically scans the dataset, extracts meaningful statistics, and generates:

1. Tables (CSV)

- Total image counts per generator, split, and class

- Sampled image-level statistics (width, height, aspect ratio, RGB means, RGB standard deviations, luminance)

2. Figures (PNG)

- Image count bar charts

- Histograms of image width and height

- Aspect-ratio category distributions

- Luminance boxplots per class

- 2D PCA projection of simple image statistics (AI vs. Nature)

3. Logs (.log)

Each run automatically creates a timestamped log file containing:

- dataset paths

- counts and sampling procedures

- warnings for unreadable images

- saved output paths

- execution progress

All logs are stored inside output/logs/.

## ðŸ§  What the Script Analyzes

For each generator, split (train / val), and class (ai / nature), the script collects:

- Image-level statistics

- Width and height

- Aspect ratio and aspect-ratio category

- Mean RGB values

- Standard deviation of RGB channels

- Approximate perceived luminance

- Dataset-level insights

- Size imbalance across generators

- Class differences in structure and color

- Typical image shapes (1:1, 4:3, 16:9, wide, tall)

- Differences visible in PCA space

The script also performs controlled sampling to avoid overloading memory when a generator contains thousands of images.

## â–¶ï¸ How to Run the Analysis

Use the following command example:
```bash
python src\analyze_genimage.py `
  --root ".\data\GenImage" `
  --generators generador1_ADM_descomprimido generador2_BigGAN_descomprimido generador3_glide_descomprimido `
  --splits train val `
  --sample_per_class 500 `
  --output_dir ".\output"
```
###### Arguments
Argument	Description:
- --root -> Root directory containing the generator folders
- --generators -> Generator folders to analyze
- --splits -> Dataset splits (default: train, val)
- --sample_per_class -> Number of images to sample per class for detailed stats
- --output_dir -> Output directory for tables, figures, and logs

## ðŸ§ª Reproducibility
